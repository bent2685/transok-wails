# 文件分享 (File Sharing)

局域网文件/文本分享：分享方在 App 维护一份分享清单，被分享方通过浏览器访问下载页拉取并下载。

## Language

**分享项 (Share Item)**:
分享清单里的一个条目。目前有文件（指向磁盘真实文件的绝对路径）、纯文本（内容内联存储、无磁盘文件、类型标记 `pure-text`）两种，即将新增共享文件夹。
_Avoid_: 文件、item

**分享清单 (Share List)**:
分享方当前对外开放的全部分享项集合。文件/文本是**快照**——添加时把信息写死进存储，被分享方通过 `POST /share/list` 一次性获取。
_Avoid_: 文件列表

**共享文件夹 (Shared Folder)**:
一种分享项，指向磁盘上一个目录根节点。其内容**不进入分享清单快照**，而是**请求时实时读取**，因此自动反映文件夹增删改。被分享方可逐层进入任意深度的子目录浏览，既可逐个下载文件，也可把某层目录整体流式打包为 zip 下载。在主 App 分享列表中以图标+名称+"文件夹"展示，不显示会过期的文件数/体积，不可在分享方本地展开浏览；除移除外另有一个"在文件管理器中打开"按钮，调用新增的后端绑定按平台（macOS `open` / Windows `explorer` / Linux `xdg-open`）打开该文件夹内部。zip 下载文件名取当前所在目录名（如 `photos.zip`）。根目录被删/移走时与现有文件一致地从清单过滤、浏览中报错退回。
_Avoid_: 目录分享

添加入口不新增按钮负担：现有"上传文件"按钮改名"上传"，点击弹小菜单二选一（选择文件 / 选择文件夹，因系统对话框无法同时选两者）；拖拽区可混合拖入文件与文件夹，后端按 `IsDir()` 自动区分。

**浏览 (Browse)**:
被分享方在共享文件夹内逐层进入子目录、查看当前目录下文件与子文件夹的动作。每次浏览触发后端实时读取目标目录，并过滤以 `.` 开头的隐藏项。请求以「分享项 Id + 相对子路径」标识位置（客户端不传绝对路径），后端用 Id 查根、拼接后做 Clean+符链解析+根包含校验。进入文件夹后搜索框仅过滤当前目录（前端本地过滤）。下载页在原页下钻并显示面包屑（共享文件夹根/子/子，可点任意一级跳回），沿用现有基于 URL hash 的页面内状态切换（现有 `#item=` 详情用同一机制），不引入路由库。列表中文件夹排在文件前、各自按名称升序。根页筛选 chip 增加 Folders（All/Folders/Files/Text）；根页"Download all"只批量下载散文件、忽略文件夹（文件夹需进入后用 zip 打包）。
_Avoid_: 列目录
